# Obsync-ICP-Backend

**ObSync on ICP** ObSync is a decentralized vault synchronization application built on the Internet Computer Protocol (ICP), designed for Obsidian users who want to sync their notes across multiple devices without trusting any third party with their data. Unlike traditional sync solutions such as Dropbox or Obsidian Sync, ObSync follows a self-hosted model — each user deploys and owns their personal ICP canister through a simple web-based onboarding flow powered by Internet Identity. No account registration, no passwords, and no centralized server ever holds user data. The developer maintains zero access to any user's vault. **How it works** All vault files are encrypted client-side using AES-256-GCM with keys derived via HKDF before any data leaves the device. The ICP canister acts as a blind, trustless storage layer — it stores and retrieves encrypted blobs without ever seeing plaintext content. Keys are generated on the user's device, never transmitted to the canister, and can be exported as a QR code or file for use on additional devices. Sync is handled by a lightweight Obsidian community plugin written in TypeScript. The plugin watches the vault folder for changes, encrypts modified files, and pushes them to the user's canister. Changes from other devices are pulled on a configurable polling interval. Conflict detection is managed using vector clocks — when the same file is edited on two devices while offline, both versions are preserved and surfaced in the plugin's conflict resolution UI for the user to review. Device tracking is built into the canister registry. Each device authenticates via its own Internet Identity delegation key, and the canister records metadata including device label, platform, last seen timestamp, and sync state. Users can name, view, and revoke devices directly from the dashboard. **Self-hosting model** Users deploy their own vault canister through a one-click web UI that calls the ICP management canister on their behalf. The deployed canister lives under the user's own Internet Identity principal — the developer has no administrative access. Users fund their canister with ICP cycles directly, costing approximately $0.43 per GiB per month on a standard 13-node subnet. A typical Markdown-heavy vault of 500MB costs under $0.25 per month to run. **Tech stack** The backend canister is written in Rust using the ic-cdk and ic-stable-structures crates, with all vault data persisted in stable memory to survive canister upgrades. The Obsidian plugin is written in TypeScript using the Obsidian Plugin API and the DFINITY agent library. Authentication is handled entirely by Internet Identity with no additional auth layer required. **Key properties** - End-to-end encrypted — the canister never sees plaintext - User-owned infrastructure — developer has zero access or control - No subscription or platform fee — users pay only for their own cycles - Works offline — Obsidian is local-first, sync happens when back online - Conflict-safe — vector clocks detect and preserve diverged versions - Auditable — canister code is publicly verifiable on-chain
